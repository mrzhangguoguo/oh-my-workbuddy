## 【知识铺垫】

（动画：GitHub 页面滑入，Yeachan-Heo/oh-my-codex 仓库亮起）

先铺个背景。Codex 圈有个神级插件包，叫 oh-my-codex，简称 OMX。

它干嘛的？

就是给 OpenAI 的 Codex CLI，外头套一层"工作流 + 技能 + 多 agent 编排"。

灵感来源你肯定熟——oh-my-zsh。

（B-roll：oh-my-zsh 装插件那种丝滑感，一行 `zsh-autosuggestions` 装好）

那问题来了：我天天用 WorkBuddy，能不能也整一个？

有人想，这还不简单？把那 46 个 markdown 文件，复制粘贴过来不就完了？

（果叔摇头，配"噔"的音效）

错。大错特错。

这就好比你看见邻居家的菜谱，照着抄一遍——结果你家压根没那口锅。

抄来的，全是死字儿。

## 【核心问题】

（动画：左边 OMX 文件树，右边 WorkBuddy 文件树，中间一道闪电隔开）

为啥不能直接抄？我扒了 OMX 全部 46 个 skill，发现它们藕断丝连地挂着四样东西。

（数据表格 —— 屏幕中央大字呈现）

| OMX 里这么写 | WorkBuddy 里该变啥 | 直接抄会怎样 |
|---|---|---|
| `$skillname` 宏调用 | `skill: xxx` 引用 | 宏不解析，全悬空 |
| `omx state write` 命令 | 任务清单 + `.workbuddy/memory` | 没这命令，直接报错 |
| `.omx/` 状态目录 | `.omw/<skill>/` 或任务清单 | 写了没人读 |
| `omx hud` / 模型路由 | 直接丢掉 | 压根没这概念 |

（口语总结，手指表格）

看见没？这表里每一样，都是 OMX 的"方言"。

你原封不动搬过来，WorkBuddy 一句听不懂。

`$deep-interview`？它不认识。
`omx state write`？命令行当场给你甩一句 "command not found"。

等于你给鱼装了轮子——它还是在地上扑腾，跑不起来。

## 【反常识解答】

（果叔靠近镜头，压低声音，背景音乐收一拍）

所以真正的活法，不是"拷贝"，是"重写"。

**首先**，宏换成引用。OMX 那套 `$foo` 互调，全改成 WorkBuddy 的 `skill: foo`。一个字不动？不行，必死。

**其次**，状态别落盘到幽灵目录。OMX 爱往 `.omx/` 写账本，WorkBuddy 这边用任务清单加每日记忆，干净利落。

**再加上**，运行时那层直接扔。OMX 的 hud 状态栏、模型路由、`omx` 二进制——这些 WorkBuddy 压根不需要，内置就有。

（案例：autopilot 闭环，动画分镜演示）

举个例子。OMX 的 `autopilot`，那套"深度访谈→规划→定目标→审查→QA"的闭环，思想贼好。

我把它整个重写进 WorkBuddy：访谈走 deep-interview，规划走 plan，执行用 Agent 工具并行编排。

跑起来，一模一样的爽。

（强调客观，字幕标注"公道话"）

说句公道话：我没复刻 OMX 的 Rust 引擎。那玩意儿解决的是 tmux 多 pane 并发，WorkBuddy 用内置 Agent 工具就干了。单 skill 不需要运行时——所以我这包，零编译，Node 一跑就完事。
